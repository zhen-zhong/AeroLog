## 简介

AeroLog 是一个自研的多端埋点平台，采用分层架构设计，支持 Android、iOS 和 Web 三个平台的 SDK。该系统的核心功能之一是调试器系统，它提供了实时监控 SDK 上报数据、配置参数 Schema、定位类型漂移和必填缺失等功能。

调试器系统通过以下方式工作：
- SDK 将事件上报到 Collector 服务
- Collector 将事件发送到 Kafka 消息队列
- Consumer 从 Kafka 消费事件并进行 ETL 处理
- API 服务提供调试接口，包括事件流和 Schema 问题检测
- Web 前端提供可视化界面进行调试和配置

**更新** 新增事件级模式验证功能，包括事件 Schema 管理界面、HMAC 签名验证、调试事件全局查看等新功能。

## HMAC 签名验证

### 签名验证机制

HMAC（Hash-based Message Authentication Code）签名验证确保事件数据的完整性和真实性，防止中间人攻击和数据篡改。

```mermaid
sequenceDiagram
participant CLIENT as "客户端 SDK"
participant SERVER as "Collector 服务"
CLIENT->>SERVER : 发送事件请求 + X-AeroLog-Signature
SERVER->>SERVER : 提取签名头部
SERVER->>SERVER : 生成期望签名
alt 签名匹配
SERVER->>CLIENT : 接受请求
else 签名不匹配
SERVER->>SERVER : 记录拒绝事件
SERVER->>CLIENT : 返回 4002 无效签名
end
```

**图表来源**
- [server/collector/internal/handler/track.go:152-165](file://server/collector/internal/handler/track.go#L152-L165)
- [server/collector/internal/handler/track.go:92-97](file://server/collector/internal/handler/track.go#L92-L97)

#### 签名生成过程

1. **签名提取**：从 `X-AeroLog-Signature` 头部获取签名值
2. **格式处理**：去除 `sha256=` 前缀和空白字符
3. **HMAC 计算**：使用 SHA-256 算法和项目密钥计算期望签名
4. **安全比较**：使用恒定时序比较函数验证签名

## 性能考虑

调试器系统在设计时充分考虑了性能优化：

### 缓冲和批处理
- SDK 使用 50 个事件的默认批次大小
- 5 秒的刷新间隔平衡了延迟和吞吐量
- 内存中的事件缓冲减少磁盘 I/O 操作

### 指数退避重传
- 最大重试间隔限制为 60 秒
- 避免网络拥塞时的级联效应
- 减少不必要的网络请求

### 数据库优化
- PostgreSQL 使用连接池管理数据库连接
- ClickHouse 优化了查询性能
- 合适的索引设计提高查询效率

### 消息队列优化
- Kafka 分区策略基于 `distinct_id` 确保同用户事件顺序
- 合适的批处理大小平衡延迟和吞吐量

### 参数预览性能优化
- **懒加载**：参数预览仅在用户需要时才加载完整数据
- **虚拟滚动**：大量参数时使用虚拟滚动技术提升性能
- **缓存机制**：已格式化的参数数据会被缓存避免重复计算
- **防抖处理**：频繁展开操作时使用防抖避免过度渲染

### 新增功能性能考虑

**事件级验证性能**
- **索引优化**：为事件定义表创建合适的索引
- **批量处理**：事件 Schema 验证采用批量处理方式
- **缓存机制**：事件定义信息在内存中缓存

**HMAC 签名验证性能**
- **异步验证**：签名验证在独立 goroutine 中执行
- **连接复用**：数据库连接池复用连接
- **超时控制**：设置合理的超时时间避免阻塞

**全局事件查看性能**
- **查询优化**：使用 OR 条件优化全局查询
- **分页机制**：限制查询结果数量避免内存溢出
- **索引利用**：合理使用数据库索引提高查询速度

## 故障排除指南

### 常见问题和解决方案

#### SDK 无法连接到服务器
1. **检查网络连接**：确保客户端可以访问 API 服务器
2. **验证 Token**：确认使用的项目 Token 正确
3. **检查 CORS 设置**：确保 API 服务器允许跨域请求

#### 事件未出现在调试器中
1. **检查 SDK 初始化**：确认 SDK 已正确初始化
2. **验证事件格式**：确保事件名称和属性符合要求
3. **检查网络状态**：确认客户端有网络连接
4. **验证 HMAC 签名**：确认客户端生成的签名正确

#### Schema 验证失败
1. **检查属性定义**：确认属性类型和必填设置正确
2. **验证枚举值**：确保提供的值在允许范围内
3. **查看事件 Schema**：检查事件级必带参数配置
4. **检查事件状态**：确认事件未被禁用
5. **查看详细错误**：检查调试器中的错误信息

#### 参数预览功能异常
1. **检查浏览器兼容性**：确保使用的浏览器支持现代 JavaScript 特性
2. **验证数据格式**：确认事件数据格式正确且不包含循环引用
3. **查看控制台错误**：检查浏览器开发者工具中的 JavaScript 错误

#### 数据库连接问题
1. **检查连接字符串**：确认数据库连接参数正确
2. **验证数据库状态**：确保数据库服务正常运行
3. **检查权限设置**：确认应用程序有足够权限

#### HMAC 签名验证失败
1. **检查签名生成**：确认客户端正确生成 HMAC 签名
2. **验证项目密钥**：确保使用正确的项目密钥

4. **查看日志**：检查服务器端日志中的详细错误信息

#### 全局事件查看问题
1. **检查权限设置**：确认用户有查看全局事件的权限
2. **验证查询参数**：确认 `include_global` 参数设置正确
3. **检查数据隔离**：确认项目数据正确隔离
