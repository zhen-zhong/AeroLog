- 采用 Next.js App Router 架构，每个分析维度（概览、事件、漏斗、留存）对应独立的 `page.tsx` 路由入口。
- 遵循“UI 组件库 + 页面逻辑”的分层模式：复用 `@/features/analytics/analytics-ui` 中的通用面板（如 `ChartPanel`, `ToolbarPanel`）与选择器，页面层仅负责状态管理（`useState`）、数据获取（`@tanstack/react-query`）及图表配置组装。
- 数据交互通过 `@/lib/api` 统一封装，利用 React Query 的 `useQuery` 和 `useMutation` 处理异步请求、缓存及加载状态。
- 图表渲染采用动态导入策略（`next/dynamic`）加载 `echarts-for-react`，确保服务端渲染（SSR）兼容性并优化首屏性能。
