- 采用 Next.js App Router 架构，每个子目录（如 `event/`, `funnel/`）对应一个独立的路由页面 (`page.tsx`)。
- 遵循“展示组件与业务逻辑分离”的设计，大量复用 `@/features/analytics/analytics-ui` 中的通用 UI 组件（如 `ChartPanel`, `ReportControls`）。
- 通过 `@tanstack/react-query` 统一管理异步数据获取，与 `@/lib/api` 封装的后端接口进行交互。
- 使用 `echarts-for-react` 实现复杂的图表渲染（趋势图、漏斗图、热力图等），并通过 `next/dynamic` 进行客户端动态导入以优化性能。
- `governance` 和 `users` 页面采用代理模式，直接导出自对应的 Feature 模块，体现了模块间的功能聚合与复用。
