- 采用 Next.js App Router 路由结构，`web/src/app/admin/` 下按业务域（events, projects, users）划分页面入口。
- 页面组件极简，仅作为路由出口，通过委托模式引用 `web/src/features/` 下的具体业务实现（如 `MetadataPage`, `ProjectsPage`）。
- 依赖共享的基础设施：使用 `@tanstack/react-query` 进行服务端状态管理，通过 `@/lib/api` 统一调用后端接口，并复用 `@/components/ui` 及 `@/features/common` 中的通用 UI 组件（如 `ProjectSelect`）。
- 数据流遵循“选择项目 -> 触发查询 -> 渲染列表/图表”的模式，各子模块独立维护本地状态但共享项目上下文。
