- **路由架构**：采用 Next.js App Router (`src/app`)，通过 `layout.tsx` 注入全局 `AppShell` 布局，实现管理后台 (`/admin`) 与分析控制台 (`/console`) 的路由隔离与共享导航。
- **状态与数据流**：在 `providers.tsx` 中集成 `TanStack Query` 作为全局数据获取与缓存层，所有业务组件通过 `src/lib/api.ts` 定义的统一客户端发起请求，实现数据逻辑与 UI 的解耦。
- **组件分层**：遵循“原子组件 (shadcn/ui) -> 布局壳 (AppShell) -> 业务特征 (features)”的分层设计，确保 UI 复用性与业务逻辑的内聚性。
