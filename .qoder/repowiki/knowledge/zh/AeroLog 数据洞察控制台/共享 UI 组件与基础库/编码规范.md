- 所有 UI 原子组件均使用 `React.forwardRef` 进行封装，并显式设置 `displayName` 以优化调试体验。
- 采用 `cn(...)` 工具函数（基于 `twMerge` 和 `clsx`）处理组件样式的条件合并，确保 Tailwind 类名冲突时的正确覆盖。
- API 客户端在 `lib/api.ts` 中集中定义，所有请求路径统一添加 `/v1` 前缀，并强制设置 `cache: 'no-store'` 以确保数据实时性。
- 业务页面组件（如 `projects-page.tsx`）遵循“桌面端表格 + 移动端卡片”的响应式设计模式，通过 Tailwind 的断点类（如 `hidden md:block`）切换展示形态。
