- **分层结构**：模块内部划分为 `components`（UI 层）、`features`（页面/功能层）和 `lib`（逻辑/工具层）。
- **UI 架构**：采用原子化设计，`components/ui` 基于 `class-variance-authority` (CVA) 和 `tailwind-merge` 封装无状态原子组件（如 Button, Card），`components/layout` 提供 `DashboardShell` 等容器组件处理全局导航与响应式布局。
- **数据流**：`lib/api.ts` 封装统一的 `fetch` 客户端，定义 TypeScript 接口并与 Go 后端 API 契约对齐；`features` 目录下的页面组件通过 `@tanstack/react-query` 消费 API 数据。
- **入口整合**：`AppShell.tsx` 作为顶层包装器，组合 `Providers` 与 `DashboardShell`，确立应用的整体视觉框架。
