- **入口与布局**：`AppShell.tsx` 作为根布局组件，包裹 `Providers` 和 `DashboardShell`。`DashboardShell` 实现了响应式侧边栏导航（桌面端固定，移动端通过 `Sheet` 抽屉展开），并定义了“报表”与“配置”两大导航分组。
- **UI 组件层**：`components/ui/` 目录包含基于 `class-variance-authority` (CVA) 和 `tailwind-merge` 构建的原子化 UI 组件（如 `Button`, `Card`, `Dialog`），支持变体（variants）和尺寸（size）定制。
- **业务特征层**：`features/` 目录组织页面级组件（如 `ProjectsPage`），遵循“Header + Metrics + Content”的布局模式，并集成 `@tanstack/react-query` 进行数据获取与状态管理。
- **基础设施层**：`lib/api.ts` 封装了统一的 `fetch` 客户端，处理 `/v1` 前缀、JSON 序列化及错误抛出；`lib/utils.ts` 提供 `cn` 工具函数用于合并 Tailwind 类名。
