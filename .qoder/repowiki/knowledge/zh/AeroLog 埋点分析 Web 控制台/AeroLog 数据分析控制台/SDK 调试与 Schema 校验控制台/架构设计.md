- **入口与状态管理**：以 `page.tsx` 为唯一客户端入口（"use client"），采用 React Hooks 结合 `@tanstack/react-query` 管理异步数据流（项目列表、事件定义、调试日志、Schema 问题）。
- **UI 架构**：基于 shadcn/ui 组件库构建双栏布局，左侧为调试过滤器与 Schema 规则编辑器，右侧通过 `Tabs` 切换展示“SDK 事件流”与“Schema 问题”数据表格。
- **数据交互层**：依赖 `@/lib/api` 模块提供的统一 API 客户端，调用 `/v1/projects/:id/debug/*` 接口获取原始调试数据，并通过 `useMutation` 实现参数类型、必填项及枚举值的持久化配置。
- **辅助逻辑**：内置 `ParamsPreview` 等本地工具组件处理复杂 Payload 的摘要展示与折叠交互，提升大数据量下的可读性。
