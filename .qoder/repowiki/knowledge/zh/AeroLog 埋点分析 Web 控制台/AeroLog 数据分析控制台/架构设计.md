- **路由架构**：采用 Next.js App Router (`web/src/app/console/`) 实现模块化页面路由，每个子目录（如 `funnel`, `retention`）对应一个独立的功能视图。
- **共享边界**：所有分析页面统一依赖 `@/features/analytics/analytics-ui` 提供的原子化 UI 组件（如 `ChartPanel`, `MetricTile`）以及 `@/lib/api` 封装的后端交互逻辑，确保了视觉风格与数据请求的一致性。
- **状态管理**：普遍使用 `@tanstack/react-query` 处理服务端状态缓存与异步数据流，结合 React Hooks 管理局部表单与筛选条件。
- **外部集成**：部分复杂功能（如元数据治理 `governance`、用户画像 `users`）通过直接引入 `@/features` 下的专用业务组件实现，体现了核心分析与周边功能的解耦设计。
