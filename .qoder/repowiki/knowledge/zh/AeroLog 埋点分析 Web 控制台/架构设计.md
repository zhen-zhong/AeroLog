- **统一应用外壳**：通过 `AppShell` 组件在根布局 (`layout.tsx`) 中注入全局导航 (`DashboardShell`) 与 React Query 提供者 (`Providers`)，确保所有子路由共享一致的布局与数据获取上下文。
- **共享 API 契约**：`lib/api.ts` 封装了针对后端 Go 服务的统一 HTTP 客户端，为管理模块（项目/元数据）与分析模块（漏斗/留存/转化）提供类型安全的数据访问层。
- **模块化路由组织**：采用 Next.js App Router 目录结构，将管理功能 (`/admin`) 与分析功能 (`/console`) 物理隔离，同时通过共享的 `components` 与 `features` 目录实现 UI 与业务逻辑的复用。
