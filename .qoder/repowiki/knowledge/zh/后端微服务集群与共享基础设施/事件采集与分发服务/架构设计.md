- 入口层：`cmd/main.go` 负责初始化 PostgreSQL 连接池、Kafka 生产者及内存缓存，并启动 Gin HTTP 服务与独立的 Metrics 监控端点。
- 业务逻辑层：`internal/handler/track.go` 实现 `/v1/track` 接口，处理请求解析、Gzip 解压、Token 鉴权及事件批量分发。
- 基础设施层：`internal/projectcache` 提供基于 PostgreSQL 的 Token 到 ProjectID 映射缓存，减少数据库查询压力；配置管理通过 `internal/config` 从环境变量加载。
- 依赖方向：核心逻辑依赖共享包 `github.com/aerolog/server/pkg` 中的模型定义、MQ 生产者及监控指标工具。
