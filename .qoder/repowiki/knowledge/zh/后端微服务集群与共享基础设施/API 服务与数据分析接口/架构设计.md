- **入口与生命周期**：`cmd/main.go` 负责初始化 PostgreSQL 连接池与 ClickHouse 客户端，配置 Gin 中间件（CORS、指标监控）并启动 HTTP 服务器。
- **配置管理**：`internal/config` 通过环境变量加载配置，支持 Postgres DSN、ClickHouse 认证及 CORS 策略。
- **业务逻辑层**：`internal/handler` 采用结构体封装处理器（如 `ProjectHandler`, `AnalyticsHandler`），直接注入数据库连接依赖。
- **数据访问模式**：Handler 内部直接执行 SQL 查询，PostgreSQL 用于元数据管理，ClickHouse 用于事件趋势、漏斗及留存等高性能分析。
