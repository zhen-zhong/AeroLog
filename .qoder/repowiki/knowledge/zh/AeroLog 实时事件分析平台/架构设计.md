- **基础设施即代码**：利用 `deploy/docker-compose.yml` 统一编排 PostgreSQL、ClickHouse、Redpanda 等核心中间件，并通过 `init/` 目录下的 SQL 脚本实现数据库 Schema 的自动化初始化。
- **跨语言工作区协同**：采用 Go Workspace (`server/go.work`) 管理多个后端微服务模块，确保共享包（如 `pkg/model`）的版本一致性；同时通过根目录 `Makefile` 提供跨前端 (Next.js) 与后端的统一开发、测试及部署指令。
- **标准化数据契约**：以 `docs/event.schema.json` 为核心协议规范，约束多端 SDK 的上报格式，并作为后端 ETL 链路（Collector -> Redpanda -> Consumer -> ClickHouse）的数据校验基准，确保全链路数据语义的一致性。
