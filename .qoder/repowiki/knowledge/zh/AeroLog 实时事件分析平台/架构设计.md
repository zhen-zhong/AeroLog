- **基础设施即代码**：根目录 `deploy/docker-compose.yml` 统一定义并启动 PostgreSQL、ClickHouse、Redpanda (Kafka)、Redis 等核心中间件，为所有子模块提供一致的运行时环境。
- **跨语言工作区管理**：后端采用 Go Workspace (`server/go.work`) 聚合 API、Collector、Consumer 及共享包 `pkg`，实现依赖本地化与原子性构建；前端与 SDK 分别通过 `package.json` 管理 TypeScript/JavaScript 生态。
- **标准化通信契约**：各子模块严格遵循 `docs` 中定义的事件 Schema 与上报协议，后端微服务间通过 Redpanda 进行异步解耦，API 服务与 Web 控制台通过 RESTful 接口交互，确保数据在采集、传输、存储及展示全链路的一致性。
- **统一可观测性体系**：基础设施层内置 Prometheus 与 Grafana，自动抓取后端各微服务的指标数据，并通过预置面板提供系统级的实时监控能力。
