- 入口层：`AeroLog` 单例对象作为唯一公开 API，负责初始化上下文、管理内存缓冲及触发定时/阈值刷新。
- 配置层：`AeroConfig` 数据类集中定义服务端地址、批处理大小、存储上限及自动追踪开关等运行时参数。
- 持久化层：基于 Room 构建 `EventDatabase`，通过 `StoredEvent` 实体和 `EventDao` 实现事件的异步增删查及容量裁剪（trimOldest）。
- 依赖方向：核心逻辑依赖 `androidx.lifecycle` 监听应用前后台切换，利用 `OkHttpClient` 执行 JSON 批量上报，并通过 `SharedPreferences` 维持匿名 ID 与用户身份的持久化状态。
