- **核心架构**：采用三阶段上报策略（内存缓冲 → IndexedDB持久化 → 指数退避重传），由 `AeroLog` 类统一调度。
- **存储层**：`EventStore` 封装 IndexedDB 操作，提供内存降级方案以兼容不支持的环境，并实现基于容量的自动淘汰机制。
- **工具层**：`utils.ts` 提供 UUID 生成、环境探测（OS/浏览器/网络）及退避算法；`types.ts` 定义跨端统一的事件协议与配置接口。
- **生命周期管理**：通过监听 `visibilitychange` 和 `pagehide` 触发 `sendBeacon` 兜底上报，确保页面卸载时数据不丢失。
