- 入口层：`cmd/main.go` 负责信号监听与应用生命周期管理。
- 编排层：`internal/app` 初始化双数据库连接池（PGX 与 ClickHouse），组装 Gin 路由并注入中间件（CORS、指标监控）。
- 业务层：`internal/handler` 采用 Handler 结构体模式，直接持有数据库连接实例处理请求。
- 数据访问：Handler 内部通过原生 SQL 直接操作 PG（元数据/配置）和 CH（海量事件日志），未引入独立的 Repository 抽象层。
- 配置层：`internal/config` 通过环境变量集中管理服务端口、DSN 及 JWT 密钥。
