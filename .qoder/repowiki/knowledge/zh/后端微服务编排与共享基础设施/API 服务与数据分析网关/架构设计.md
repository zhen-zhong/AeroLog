- **入口与生命周期**：`cmd/main.go` 负责信号监听与应用启动，`internal/app/app.go` 封装 HTTP 服务器、指标服务及数据库连接池（PostgreSQL 和 ClickHouse）的生命周期。
- **路由与中间件**：在 `app.go` 中统一配置 CORS、Recovery 及请求耗时/总量监控中间件，并通过 `/v1` 分组注册各业务 Handler。
- **Handler 层设计**：采用扁平化结构，所有 Handler（如 `ProjectHandler`, `AnalyticsHandler`）直接注入 `pgxpool.Pool` 和 `driver.Conn`，在方法内直接编写 SQL/ClickHouse 查询逻辑，未引入独立的服务层或仓储层。
- **双存储架构**：通过 `internal/config/config.go` 管理多源配置，业务逻辑根据需求选择 PostgreSQL（项目、事件定义、身份映射）或 ClickHouse（事件流、用户画像、漏斗分析）。
