- **协议驱动解耦**：以 `docs` 定义的 JSON Schema 为唯一真理源，确保 Android/iOS/Web SDK 与后端 Collector/Consumer 之间的数据语义一致性。
- **异步流式架构**：后端采用 Collector -> Kafka -> Consumer 的流式处理链路，实现高并发采集与 ETL 逻辑的物理隔离及弹性伸缩。
- **统一基础设施编排**：通过 `infra_deploy` 模块的 Docker Compose 配置，一键拉起 ClickHouse、Redpanda、PostgreSQL 等核心组件，为所有业务模块提供标准化的本地开发环境。
- **前后端分离协同**：Web 控制台通过 API 服务获取经 ClickHouse 聚合后的数据，实现从数据采集到可视化分析的闭环。
