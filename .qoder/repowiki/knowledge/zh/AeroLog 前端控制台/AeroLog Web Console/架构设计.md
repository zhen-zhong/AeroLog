- 采用 Next.js 14 App Router 架构，通过 `AppShell.tsx` 实现全局布局（Header/Sider/Content）。
- 使用 Ant Design (`ConfigProvider`, `Layout`) 构建 UI 框架，集成 TanStack Query 进行服务端状态管理。
- 封装统一的 API 客户端 (`lib/api.ts`)，通过环境变量 `NEXT_PUBLIC_API_BASE` 指向 Go 后端服务，并配置了 Next.js 的 `rewrites` 以支持开发环境下的跨域代理。
- 依赖 ECharts 进行数据可视化渲染，Zustand 用于轻量级状态管理。
