- **路由与布局编排**：采用 Next.js App Router (`src/app`) 作为顶层路由入口，`layout.tsx` 注入 `AppShell` 提供统一的 Dashboard 侧边栏与顶部导航，实现 `admin`（管理）与 `console`（分析）两大业务域的视觉与交互一致性。
- **数据流与状态协同**：通过 `src/lib/api.ts` 封装统一的 RESTful 客户端，所有子模块共享该层与后端 Go 服务通信；利用 `@tanstack/react-query` (`providers.tsx`) 处理服务端状态的缓存与同步，结合 `Zustand` (`stores/`) 管理跨页面的项目上下文切换与 UI 偏好，实现数据获取与本地状态的解耦。
- **组件化架构**：遵循 Feature-Sliced Design 思想，将通用原子组件 (`components/ui`)、业务布局壳 (`components/layout`) 与领域功能模块 (`features/`) 分层隔离，确保 `admin` 与 `console` 在复用基础 UI 能力的同时保持业务逻辑的独立性。
