- **路由编排**：采用 Next.js App Router (`web/src/app/console/`) 实现模块化页面路由，每个子目录（如 `funnel/`, `retention/`）对应一个独立的功能页面。
- **共享 UI 契约**：所有分析页面统一依赖 `@/features/analytics/analytics-ui` 提供的标准化组件（如 `AnalyticsHeader`, `ChartPanel`, `MetricTile`），确保了跨模块的视觉一致性与交互规范。
- **全局状态同步**：通过 `useProjectStore` (Zustand + localStorage) 实现跨页面的项目上下文共享，确保用户在切换分析维度时保持当前选中的项目 ID。
- **数据层抽象**：各模块通过统一的 `api` 客户端与后端交互，利用 `@tanstack/react-query` 进行声明式数据获取、缓存管理及异步状态处理。
