- 所有 UI 组件均使用 `React.forwardRef` 进行包装，并通过 `displayName` 显式声明组件名称以优化调试体验。
- 样式合并统一采用 `cn` 工具函数（结合 `clsx` 与 `tailwind-merge`），确保 Tailwind 类名冲突时的正确覆盖。
- API 响应遵循统一的 `{ data: T }` 或 `{ data: T[] }` 包装结构，客户端通过 `ApiOne` 和 `ApiList` 泛型接口进行类型约束。
- 业务页面（Features）普遍采用“桌面端表格 + 移动端卡片”的双轨渲染模式，通过 `hidden md:block` 等 Tailwind 断点实现响应式切换。
