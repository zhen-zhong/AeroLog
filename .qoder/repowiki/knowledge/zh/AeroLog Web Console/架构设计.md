- **统一应用壳层**：通过 `AppShell` 组件组合 `Providers`（集成 TanStack Query）与 `DashboardShell`，为 `admin` 和 `console` 子模块提供一致的布局、导航及数据请求上下文。
- **集中式 API 网关**：`src/lib/api.ts` 封装了所有后端交互逻辑，统一管理项目、事件、用户及各类分析接口（如漏斗、留存），确保各功能模块通过统一的 `/v1` 路径访问 Go 后端服务。
- **模块化路由组织**：采用 Next.js App Router 结构，将管理职能（`/admin`）与分析职能（`/console`）在物理路径上隔离，但共享 `src/components` 中的基础 UI 库与业务组件（如 `project-select`）。
