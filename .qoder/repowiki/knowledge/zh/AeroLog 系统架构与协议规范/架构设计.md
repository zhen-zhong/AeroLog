- **分层架构**：采用 SDK → Collector (Go/Gin) → Kafka → Consumer (Go/ETL) → ClickHouse/Postgres → API (Go) → Next.js 的数据流水线，详见 `docs/architecture.md`。
- **协议规范**：定义基于 HTTP POST 的上报协议 (`/v1/track`)，支持 gzip 压缩、HMAC 签名鉴权及批量上报，详见 `docs/protocol.md`。
- **数据契约**：使用 JSON Schema (`docs/event.schema.json`) 严格约束事件结构，区分预置属性（`$` 前缀）与自定义属性，支持 track 及 profile 操作类型。
- **可观测体系**：规定 Collector、Consumer、API 服务独立暴露 Prometheus metrics 端口（9101-9103），并定义核心业务指标（QPS、Kafka Lag、DLQ 数量等），详见 `docs/observability.md`。
