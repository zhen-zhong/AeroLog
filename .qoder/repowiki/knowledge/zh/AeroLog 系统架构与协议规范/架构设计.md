- **数据流架构**：采用 SDK → Collector (Go/Gin) → Kafka → Consumer (Go/ETL) → ClickHouse/Postgres → API (Go) → Next.js 的分层设计，详见 `docs/architecture.md`。
- **协议规范**：定义统一的 HTTP 上报接口 `/v1/track`，支持批量 gzip 压缩与 HMAC 签名校验，兼容神策协议，详见 `docs/protocol.md`。
- **数据模型**：通过 `docs/event.schema.json` 严格约束事件结构，区分预置属性（`$` 前缀）与自定义属性，支持 track 及 profile 系列操作。
- **可观测体系**：规定 Collector/Consumer/API 独立暴露 Prometheus metrics 端口（9101-9103），并定义核心业务指标与告警规则，详见 `docs/observability.md`。
