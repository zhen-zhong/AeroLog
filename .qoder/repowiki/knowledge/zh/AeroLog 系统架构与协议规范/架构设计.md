- **分层架构**：采用 SDK → Collector (Go/Gin) → Kafka → Consumer (Go/ETL) → ClickHouse/Postgres → API (Go) → Next.js 的数据流水线。
- **协议规范**：通过 `protocol.md` 和 `event.schema.json` 统一定义了基于 HTTP/gzip 的上报接口及 JSON Schema 校验规则，支持多端（Web/Android/iOS）标准化接入。
- **可观测体系**：在 `observability.md` 中确立了基于 Prometheus + Grafana 的监控方案，约定各 Go 服务独立暴露 metrics 端口（910x 系列）并定义了核心业务指标（如 QPS、Kafka Lag、DLQ 数量）。
- **演进策略**：明确了从单机 Docker Compose (MVP) 到分布式 ClickHouse/Flink (大规模) 的容量演进路径及 WAL 兜底机制。
