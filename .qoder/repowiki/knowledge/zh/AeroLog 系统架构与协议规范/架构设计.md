- **数据流架构**：采用 SDK → Collector (Go/Gin) → Kafka → Consumer (Go/ETL) → ClickHouse/Postgres → API (Go) → Next.js 的分层处理链路，详见 `docs/architecture.md`。
- **协议规范**：定义基于 HTTP POST 的上报协议 (`/v1/track`)，支持 gzip 压缩与 HMAC 签名校验，兼容神策协议，详见 `docs/protocol.md`。
- **数据模型**：通过 `docs/event.schema.json` 定义严格的 JSON Schema，约束事件类型、预置属性（如 `$os`, `$lib`）及用户身份标识（`distinct_id`, `anonymous_id`）。
- **可观测体系**：规定 Collector、Consumer、API 三个 Go 服务独立暴露 Prometheus metrics 端口（9101-9103），并定义核心监控指标与告警规则，详见 `docs/observability.md`。
